1.2 深度代码阅读（三圈，2.5小时）

这是Phase 1最重要的部分。我们用"三圈法"逐步深化对代码的理解。每圈都专注于不同的问题。

🎯 学习目标

关键问题： Agent执行的大致流程是什么？一次迭代循环包含哪几个步骤？

  • 评估的逻辑在哪？
  • 保留/回滚的逻辑在哪？

## AutoResearch Code Deep Dive - 第1圈：流程理解

### Agent execution loop (伪代码)

```python
while not converged:
    # 1. 基于历史，生成代码修改提案
    modification = agent.propose_modification(history)
    
    # 2. 应用修改
    apply_change(modification)
    
    # 3. 训练并测试
    result = train_and_test(duration=5min)
    
    # 4. 评估是否改进
    improved = result.score > best_score
    
    # 5. 保留或回滚
    if improved:
        keep_change(modification)
        best_score = result.score
    else:
        revert_change(modification)
```

📚 第2圈：关键细节 (60min)

关键问题： 每个步骤的具体实现细节是什么？

深入理解这些问题：

  • 怎样"修改"代码？随机选择？启发式选择？
  • 怎样"评估"改进？用什么指标？CORE分数？
  • 怎样"决策"保留/丢弃？简单的阈值比较？
  • 如何避免无限循环？有convergence条件吗？
  • 如何处理训练失败的情况？异常恢复机制？

📚 第3圈：映射到VEX场景 (50min)

关键问题： AutoResearch的原理如何应用到VEX的Agent中？

做这些对比：

  • AutoResearch的"train"功能 ← → VEX Designer生成步骤（类似吗？）
  • AutoResearch的"evaluate"功能 ← → VEX Integrator检查步骤有效性（类似吗？）
  • AutoResearch的"modify"策略 ← → VEX中什么情况下需要改？
  • 如果要用到VEX中，需要改什么？
💡 学习建议：

三圈循环进行。第1圈粗读，找到主要文件和函数。第2圈细读，理解每个步骤的细节。第3圈思考，不读代码，而是思考"这个在VEX中怎么用"。

1.3 理论总结 (30min)

把你学到的东西浓缩成核心原则。这些原则将直接指导Phase 3的集成设计。

🎯 学习目标

提炼AutoResearch的5个核心原则，并思考它们在VEX中的应用。

📝 产出：autoresearch-principles.md

AutoResearch的5个核心原则（你的理解）：

原则1：原子性改动
- 每次修改一个可独立评估的单元
- VEX应用：Designer改动步骤顺序时，每次改动应该是...?

原则2：快速反馈循环
- 训练5分钟 → 即刻反馈
- VEX应用：组装步骤生成的快速验证应该...?

原则3：历史感知
- Agent记住之前的尝试，避免重复
- VEX应用：Designer应该记住...?

原则4：保守决策
- 只有明确改进才保留
- VEX应用：什么情况下算"明确改进"？

原则5：可解释性
- 能说出为什么保留/丢弃
- VEX应用：生成的指南要能解释为什么这样设计

💡 学习建议：

这不是复述官方文档，而是你自己用语言总结的原则。用自己最直白的语言写，越简单越好。这样写出来的东西才是真正理解了。
